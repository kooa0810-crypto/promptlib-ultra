1. INTRODUCTION & PHILOSOPHY
================================================================================

Welcome to Prompt Forge. This database contains over 100,000 procedurally 
generated "Master Prompts." Unlike simple one-line queries, Master Prompts are 
engineered structures designed to extract the highest quality, most consistent, 
and most useful outputs from Large Language Models (LLMs).

Why Master Prompts?
- Consistency: Reduces hallucination and random variance.
- Depth: Forces the AI to think step-by-step (Chain of Thought).
- Structure: Ensures output is formatted exactly how you need it.

2. THE ANATOMY OF A MASTER PROMPT
================================================================================

Every prompt in this database follows a strict architectural framework. 
Understanding this structure will help you customize them effectively.

A. ROLE DEFINITION
   - Assigns a specific persona (e.g., "Senior Python Developer").
   - Primes the AI's latent knowledge base for that specific domain.

B. TASK OBJECTIVE
   - A clear, imperative statement of what needs to be done.
   - Uses active verbs (e.g., "Analyze," "Write," "Debug").

C. CONTEXT & AUDIENCE
   - Provides background information necessary for nuance.
   - Defines who the output is for (e.g., "Beginners," "CEOs").

D. CONSTRAINTS & RULES
   - Sets boundaries (e.g., "No jargon," "Max 500 words").
   - Prevents common AI pitfalls like verbosity or vagueness.

E. OUTPUT STRUCTURE
   - Dictates the exact format (e.g., Markdown, Table, JSON).
   - Ensures the result is ready to use without reformatting.

5. THE MASTER PROMPT GENERATOR GUIDE
================================================================================

If you can't find the exact prompt you need, use the built-in Generator.

Step 1: Define the Role
   - Be specific. Instead of "Writer," use "Direct Response Copywriter."

Step 2: Define the Task
   - Be actionable. Instead of "Help me with code," use "Debug this function."

Step 3: Add Context
   - Who is this for? What is the background? Why does this matter?

Step 4: Set Constraints
   - Word count? Tone? Format? Specific things to avoid?

Step 5: Generate
   - The tool will assemble these into a structured Master Prompt using the
     anatomy defined in Section 2.

6. BEST PRACTICES FOR LLM INTERACTION
================================================================================

1. BE SPECIFIC
   Vague inputs yield vague outputs. Provide as much detail as possible.

2. PROVIDE CONTEXT
   The AI doesn't know your business or your life. Tell it what it needs to know.

3. ITERATE
   Treat the AI as a partner. If the first output isn't perfect, reply with:
   "Make it more concise," or "Expand on point 3."

4. USE EXAMPLES (Few-Shot Prompting)
   If you have a specific style in mind, paste an example of good output into
   the prompt and say: "Follow this style."

5. ASSIGN ROLES
   Always start with "Act as a [Expert Role]." This significantly improves quality.

7. TROUBLESHOOTING & REFINEMENT
================================================================================

Problem: The AI is too verbose.
Solution: Add constraint: "Be concise. Use bullet points. Max 200 words."

Problem: The AI is hallucinating facts.
Solution: Add constraint: "Only use information provided in the context. If you
don't know, state that you don't know."

Problem: The tone is wrong.
Solution: Explicitly define the tone: "Tone: Professional, empathetic, and direct."

Problem: The format is messy.
Solution: Specify the output format: "Output as a Markdown table with columns X, Y, Z."
